# Documentation

The project documentation is deliberately kept beside the code so that every
version of the package has matching explanations.

- [README](../README.md): installation, beginner tutorial, model selection,
  interpretation, limitations, and development commands.
- [Algorithm guide](algorithm.md): the explicit-duration model, shifted
  Poisson durations, forward-backward recursion, EM updates, and Viterbi
  decoding.
- [API reference](api.md): constructor parameters, learned attributes, methods,
  shapes, and exceptions.
- [Examples](../examples): small scripts that can be run from a checkout.
- [Contributing](../CONTRIBUTING.md): local setup and pull-request expectations.
- [Releasing](../RELEASING.md): the maintainer's manual pre-release checklist.

The README is the best starting point for a first-time HSMM user. The algorithm
guide is the next step when you want to connect the implementation to the
underlying probability model.
